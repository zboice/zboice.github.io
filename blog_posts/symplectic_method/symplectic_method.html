Now that we have developed this symplectic method, it's valuable to compare it to other integration methods to understand its performance and strengths. The video below compares the symplectic method to two numerical integrators from scipy, all with the same, fixed step size.

As the simulation continues, we clearly see that the symplectic integrator conserves energy and keeps the orbiting particle on its elliptical path. But this isn't exactly a fair competition among the integrators. The strength of the Runge-Kutta method is how well it can be tuned to the problem to produce highly accurate results blazingly fast. So now, let's integrate for the same exact conditions, but with absolute and relative error tolerance set for the two non-symplectic methods.
Now, we see that non-symplectic methods achieve a better match with the Hamiltonian at early times. At late times, error compounds and we see that the symplectic method again reigns supreme. If we run any position or energy comparison for long enough, the symplectic method will win out in accuracy. But at what cost? The table below shows how long each method took for the above simulation, and how many calls each made to the system derivative function (often the most computationally intensive step).

Method Time elapsed Function calls
Symplectic 2.1131s 258478
RK23 0.1969s 13937
LSODA 0.0197s 1605

This still isn't a perfect comparison. The scipy code is optimized to work blazingly fast whereas I wrote my code in about 20 minutes, with no real optimizations. Of course that means I'll keep working to optimize my code, but it's unlikely to create a 100x speed up so that the symplectic integrator is beating LSODA. In the real world, people use symplectic integrators for many body gravitational systems or for situations where a small drift of function values could have catastrophic effects. So, it's definitely still worth understanding symplectic integrators, but part of that understanding is knowing when not to use one.
